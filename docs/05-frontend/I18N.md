# Frontend — i18n

Bilingual **Albanian (al)** + **English (en)**, matching ops-core's locales. Albanian is the primary venue language; English for international organizers.

## Mechanics
- `useLocaleStore` (Zustand): `locale`, `setLocale`; persists to `localStorage('pyramid.locale')`. Resolution: `?lang=` → localStorage → `navigator.language` → `en`.
- `useT()` returns `t(key, params?)`. Missing key → dev warning + returns the key.
- Strings live in `i18n/al.json` + `i18n/en.json`. **Key parity is enforced** (both files define the same keys; a test fails on drift) — mirrors ops-core's locale parity rule.
- The API client sends `Accept-Language` from the store so ops-core returns localized `message` for each `messageKey`.

## Design implications (carry into Claude Design)
- **Albanian runs ~20–30% longer** than English. Buttons, chips, table headers, and the sidebar must flex without truncation or wrapping that breaks layout. Design every artboard in both locales.
- Dates/times: display in **Europe/Tirane** wall-clock (the wire is UTC); format per locale. Money: `ALL` with locale-aware thousands grouping (`134.000 L` / `134,000 ALL`), value stays integer minor units.
- Numbers are tabular in both locales.
